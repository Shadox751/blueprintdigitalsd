# Roadmap: Blueprint Digital

## Overview

Blueprint Digital goes from zero assets to a running solo agency selling contractor websites and AI receptionists in San Diego. The roadmap sequences around one hard constraint: Allan is going out cold tomorrow morning. Phase 1 is tonight's work -- business foundation and sales tools so he walks in with a card, a contract, a price, and a trained pitch. Phase 2 builds the GoHighLevel delivery platform (days 2-5) so there is a real demo and tested templates before any deal closes. Phase 3 activates only after the first close -- no delivery infrastructure built on spec. Phase 4 keeps clients paying month after month. Phase 5 turns logged visits into a compounding sales pipeline.

## Phases

**Phase Numbering:**
- Integer phases (1, 2, 3): Planned milestone work
- Decimal phases (2.1, 2.2): Urgent insertions (marked with INSERTED)

Decimal phases appear between their surrounding integers in numeric order.

- [ ] **Phase 1: Ready Tonight** - Business identity, contract, pricing, sales commands, and route for tomorrow morning

- [ ] **Phase 3: GoHighLevel Platform** - Account setup, master templates, Voice AI config, demo site, SMS registration
- [ ] **Phase 4: Client Delivery Pipeline** - Intake-to-handoff workflow triggered by first close
- [ ] **Phase 5: Client Retention System** - Monthly reports, follow-up sequences, and value proof for retainer justification

## Phase Details

### Phase 1: Ready Tonight
**Goal**: Allan walks out the door tomorrow morning with a business identity, a contract in his pocket, a price sheet in his head, a trained pitch, and a route planned
**Depends on**: Nothing (first phase)
**Requirements**: FOUND-01, FOUND-02, FOUND-03, FOUND-04, SALES-01, SALES-02, SALES-04, SALES-07
**Success Criteria** (what must be TRUE):
  1. Allan can hand someone a business card with Blueprint Digital branding, his name, phone number, and website
  2. Allan can present a one-page contract with numbered deliverables and 50% deposit terms when a contractor says yes
  3. Allan can state his pricing confidently -- $1,200 setup + $200/mo standard, $1,000 + $150/mo floor -- without hesitation
  4. Allan can run `/brief` for any business and get talking points before walking in, run `/pitch` for a contractor-type-specific script, and run `/route` for tomorrow's optimized route from Chula Vista
  5. Allan can practice a full contractor conversation via `/roleplay` before his first visit

### Phase 2: Sales Operator System
**Goal**: Allan can log every visit, handle any objection with a trained response, track his full pipeline, and review weekly performance -- all from slash commands
**Depends on**: Phase 1
**Requirements**: SALES-03, SALES-05, SALES-06, LOG-01, LOG-02, LOG-03, LOG-04
**Success Criteria** (what must be TRUE):
  1. After each visit, Allan can run `/log` and the visit is appended to today's daily log file with business name, address, contact, outcome, and objections
  2. Allan can run `/objection` with any objection he heard and get counter-arguments and practice prompts back
  3. Allan can run `/dashboard` and see a pipeline summary -- total visits, follow-ups pending, deals closed, revenue
  4. At end of week, a summary exists showing visits made, conversion rate, top objections, and revenue pipeline
  5. Claude work log tracks what was built or changed each session

### Phase 3: GoHighLevel Platform
**Goal**: A tested, cloneable contractor website template and configured Voice AI exist in GoHighLevel with a live demo site Allan can show on his phone during pitches

  4. A2P 10DLC SMS registration is submitted (takes 1-2 weeks -- must be started early)

### Phase 4: Client Delivery Pipeline
**Goal**: When a contractor says yes, Allan has a repeatable intake-to-live workflow that delivers a website and AI receptionist without custom engineering per client
**Depends on**: Phase 3 (templates must exist before first client delivery)
**Requirements**: WEB-01, WEB-02, WEB-03, WEB-04, WEB-05, RECEP-01, RECEP-02, RECEP-03
**Success Criteria** (what must be TRUE):
  1. Allan can run `/intake` and capture all info needed for a contractor site in one sitting (business name, services, area, phone, photos, reviews)
  2. Allan can run `/build` with intake data and get complete website copy, page structure, and GHL configuration output ready to paste
  3. Allan can run `/launch` and follow a checklist to go live with domain, DNS, and Google Business Profile link
  4. Allan can run `/handoff` and generate a client onboarding doc explaining how to send photos, update info, and what to expect
  5. Allan can run `/recep` to configure Voice AI for the specific client with their business name, services, and booking script
